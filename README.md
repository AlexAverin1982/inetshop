# Курсовая работа "Приложение для анализа транзакций"
## Цель домашней работы
Используя навыки, полученные в ходе обучения, разработать приложение, анализирующее данные и выводящее 
результаты анализа, а так же провести тестирование написанного кода

## Возможности приложения
+ Вывод сводной информации о тратах и кэшбэке по отдельным банковским счетам за указанный период;
+ Вывод отсортированной информации о наибольших тратах в разрезе категории за указанный период;
+ Курс валют, интересующих пользователя;
+ Цены на акции, интересующие пользователя;
+ Суммарные расходы за указанный период;
+ Отсортированные расходы по категориям за указанный период;
+ Отдельные расходы на переводы и обнал;
+ Суммарный приход за указанный период;
+ Инструмент, показывающий сколько можно накопить в инвесткопилке, округляя расходы до определенного предела;
+ Отчет о средних расходах в будни и выходные дни за последние три месяца до указанной даты.

## Структура проекта
#### + Исходный код функционала приложения находится в папке `src`
  + модуль `main` - точка входа приложения, отображает главное меню приложения для навигации по компонентам программы;
  + модуль `views` - пункты главного меню приложения, так называемые страницы, основная часть пользовательского 
    интерфейса, служащая для: 
    + запроса информации, которую должен ввести пользователь;
    + вывода результатов работы выбранных компонентов приложения;
  + модуль `menus` - служебный модуль, содержащий меню отдельных компонентов приложения, дополнительные компоненты 
    интерфейса пользователя;
  + модуль `utils` содержит подпрограммы работы с датафреймами и небольшими утилитами для упрощения работы с данными;
  + модуль `datetime_utils` содержит инструменты для облегчения работы с датами и временем, включая компоненты 
    интерфейса пользователя для запроса специфических данных;
  + модуль `decorators` содержит инструмент для журналирования работы отдельных компонентов программы и записи данных в 
    файл по указанному пути;
  + модуль `currencies_and_stocks_utils` содержит подпрограммы для обработки и получения данных о валютах и акциях, 
    исключая инструменты, обращающиеся к внешним ресурсам;
  + модуль `external_api` содержит инструменты, обращающиеся к внешним ресурсам для получения данных о валютах и акциях;
  + модуль `reports` содержит подпрограмму, формирующую выбранные в задании отчетные данные. Реализованный функционал
    планируется размещать в разделе "Отчеты" 
  + модуль `services` содержит подпрограмму, формирующую выбранные в задании бонусные данные. Реализованный функционал
    планируется размещать в разделе "Сервисы" 
  
#### + Тесты работы исходного кода находится в папке `tests`
#### + Данные, с которыми будет работать приложение находятся в папке `data`
По условиям задания, данные хранятся в формате `xlsx`
#### + Кэш данных, полученных от внешних ресурсов (апи) для уменьшения числа обращения к апи, находится в папке cache
Данные кэша хранятся в формате `json`. Каждое принудительное обновление данных через внешние ресурсы перезаписывает 
кэш полностью.
#### + Журналы работы компонентов приложения находятся в паке logs
Журнал `log.log` хранит записи декоратора журналирования без указания имени журнала, это журнал по умолчанию.
#### + Данные отчета о покрытии кода тестами находятся в папке htmlcov
Основновная информация доступна через файл `index.html`



## Инструкции по использованию приложения
- Запустите модуль `src\main.py`, следуйте инструкциям на экране
- Отдельными компонентами можно пользоваться, импортируя их из соответствующих модулей папки `src`
